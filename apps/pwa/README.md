## GitHub Pages 免费发布准备

项目已支持部署到 GitHub Pages 的仓库子路径：

- 路由使用 Hash 模式，刷新页面不会因为 `/player/...` 子路由导致 404。
- 数据、图片、音频和讲稿路径会自动跟随 Vite 的 `BASE_URL`。
- GitHub Actions 发布时会自动设置 `VITE_BASE_PATH=/<仓库名>/`。
- PWA manifest 与 service worker 已适配 GitHub Pages 子路径。

发布步骤：

1. 在 GitHub 创建公开仓库。
2. 将当前项目提交并推送到 `main` 分支。
3. 进入仓库 `Settings -> Pages`，Source 选择 `GitHub Actions`。
4. 推送后等待 `Deploy PWA to GitHub Pages` 工作流完成。
5. 访问 `https://<用户名>.github.io/<仓库名>/`。
